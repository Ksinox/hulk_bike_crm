# Дизайн-система Халк Байк CRM

Ориентир для всех новых экранов и компонентов. Если добавляешь новый паттерн —
сначала ищи его здесь. Если паттерна нет — заведи, затем используй.

---

## 1. Базовые принципы

1. **Плотность информации важнее декораций.** Владелец читает карточки
   десятками в день — приоритет у цифр и статусов, не у иллюстраций.
2. **Одна поверхность — одна задача.** Никаких вложенных карточек внутри
   карточек. Секции разделяются пробелом и подзаголовком, а не обводкой.
3. **Цвет = смысл, а не украшение.** Зелёный — норма, красный — проблема,
   синий — акцент/действие, оранжевый — предупреждение / возврат.
4. **Одинаковые сущности — одинаково.** Клиент выглядит везде одинаково:
   аватар / фото → имя → рейтинг. Аренда: статус → скутер — клиент → телефон.
5. **Действия — справа, контекст — слева.** В шапке карточки слева
   читаемая инфа, справа — основное действие и dropdown остальных.

## 2. Типографика

Шрифты подключены из Google Fonts в `apps/web/src/index.css`.

| Назначение | Семейство | Пример использования |
|---|---|---|
| Основной текст (body, data) | **Inter** 400/500/600/700 | названия полей, значения, списки |
| Заголовки, крупные метрики | **Manrope** 800 (класс `.font-display`) | H1, KPI-значения |
| Числовые поля | `tabular-nums` (feature `tnum`) | суммы, телефоны, рейтинги |

### Масштаб размеров

| Назначение | Размер | Вес |
|---|---|---|
| H1 страницы (Клиенты / Аренды) | `text-[34px]` | 800 Manrope |
| H2 карточки (имя клиента, скутер) | `text-[22–24px]` | 800 Manrope |
| Подзаголовок секции (uppercase) | `text-[11px] tracking-wider` | 600 Inter, `text-muted-2` |
| Значения полей | `text-[13–16px]` | 600–700 Inter |
| Hint / подпись | `text-[10–11px]` | 500 Inter, `text-muted-2` |
| KPI-значение | `text-[16–20px]` | 800 Manrope |

Заголовки **никогда не bold+ink-цвет+крупный+uppercase одновременно** —
выбери одно выделение и держись его.

## 3. Цветовая палитра

Все цвета объявлены HSL-компонентами в `:root` (`index.css`) и замаплены в
Tailwind через `tailwind.config.ts`. Всегда используй токены, **не хардкодь
hex**.

### Нейтральные

| Токен | Применение |
|---|---|
| `bg` | фон страницы |
| `bg-grad-1/2` | мягкий градиент фона |
| `surface` | фон карточек (белый) |
| `surface-soft` | фон «вложенных» блоков, hover-состояния, таблицы |
| `border` | обычные границы |
| `border-strong` | чуть более заметные границы |
| `ink` | основной текст |
| `ink-2` | подзаголовки, контент |
| `muted` | второстепенный текст, подписи |
| `muted-2` | очень мелкие подписи, плейсхолдеры |

### Бренд-синий

| Токен | Когда |
|---|---|
| `blue-50` | фон выделенных плашек/чипов (мягкий акцент) |
| `blue-100` | обводка акцента |
| `blue-600` | **primary** — кнопки, иконки, ссылки hover, активная строка |
| `blue-700` | hover/pressed для primary |
| `blue` | резерв |

### Семантические

| Ситуация | Фон | Текст |
|---|---|---|
| Успех / норма / активно | `green-soft` | `green-ink` |
| Проблема / просрочка / ЧС | `red-soft` | `red-ink` |
| Предупреждение / возврат сегодня / ущерб | `orange-soft` | `orange-ink` |
| Информационное / служебное | `purple-soft` | `purple-ink` |
| Нейтральное / прошлое / отменено | `surface-soft` | `muted` |

Насыщенные токены (`red`, `green`, `orange`, `purple`) — только для рамок,
точек-индикаторов, прогресс-полос. **Не для заливок** — используй `-soft`.

## 4. Радиусы и форма

Все радиусы — в `--r-*` и на уровне Tailwind `rounded-{sm|md|lg|xl|2xl}`.

| Элемент | Радиус |
|---|---|
| Инпуты, пилюли, чипы | `10px` (`rounded-[10px]`) |
| Кнопки (обычные) | `full` |
| Кнопки (большие / primary крупные) | `rounded-[14px]` |
| Карточки-блоки | `rounded-[14px]` |
| Экранные контейнеры, модалки | `rounded-2xl` (24px) |
| Строки таблицы / чип в сетке | `rounded-[12px]` |

**Фото клиента** — особый случай: `rounded-[14px]`, соотношение `9:16`
(как паспортное фото). Всегда через компонент `ClientPhoto`.

## 5. Тени

Три уровня в `@layer utilities` (`index.css`):

| Класс | Когда |
|---|---|
| `shadow-card-sm` | карточки в потоке, строки списка |
| `shadow-card` | карточка hover, выпадающие меню |
| `shadow-card-lg` | модалки, активная строка списка, popover’ы |

Правило: **тень повышается только при активации** (hover/selected/модалка).
Обычная карточка — `shadow-card-sm`.

## 6. Раскладка и отступы

- Корневой контейнер приложения: `max-w-[1440px]`, `p-[18px]`, `gap-[18px]`
- Между карточкой-блоком и содержимым: `p-5` (20px)
- Между секциями внутри карточки: `gap-3` (12px) или `gap-4` (16px)
- Между мелкими элементами в строке (чипы, dot-разделители): `gap-1.5`/`gap-2`
- Таб-бар: `border-b border-border`, активная — `border-b-2 border-blue-600`

### Split-view (список слева + карточка справа)

Используется на страницах «Клиенты» и «Аренды»:

```tsx
<div className="grid flex-1 gap-4 lg:grid-cols-[360px_1fr]">
  {/* или [420px_1fr] для аренд */}
  <List />
  <DetailCard />
</div>
```

## 7. Компоненты и паттерны

### Карточка сущности (клиент / аренда)

```
[фото / иконка]   Имя/название + бейджи статуса        [actions]
                  инфо-строка: id · тариф · рейтинг · источник · телефон
                  (телефон — первым, крупный)
─────────────────────────────────────────────────────
KPI-чипы (4–5 шт в ряд)
─────────────────────────────────────────────────────
Табы: Условия · Платежи · Возврат · Инциденты · Задачи · Документы
[контент активной табы]
```

Правила:
- Иконки в заголовке (скутер, клиент) — **не синие**, серого `ink-2` цвета.
  Подчёркивание появляется только на hover.
- Рейтинг клиента привязан к его имени, не в общей инфо-строке.
- Статус-пилюля **перед** названием.

### Action bar + dropdown

Правило: **одна primary-кнопка** слева, остальное — в dropdown `Действия ▾`
справа. Никогда не выкладывай 5 кнопок в ряд.

```tsx
<RentalActionsMenu actions={actions} onAction={handleAction} />
```

Тона кнопок:
- `primary` — `bg-blue-600 text-white`
- `warn` — `bg-orange-soft text-orange-ink`
- `danger` — `bg-red-soft text-red-ink`
- `ghost` — `bg-surface-soft text-ink-2`

### KPI-чип

```tsx
<div className="rounded-[12px] bg-blue-50 px-3 py-2">
  <div className="text-[10px] font-semibold uppercase tracking-wider text-muted-2">
    Оборот
  </div>
  <div className="mt-0.5 font-display text-[20px] font-extrabold leading-none text-ink">
    12 000 ₽
  </div>
  <div className="mt-0.5 text-[10px] text-muted-2">за всё время</div>
</div>
```

Фон чипа по тону: `bg-blue-50` (нейтральный/информационный),
`bg-green-soft/60` (успех), `bg-red-soft/60` (проблема), `bg-surface-soft`
(серый/неактивный).

Для составного значения (несколько строк внутри) — `KpiChipSplit`
с массивом `rows: { key, value }[]`.

### Статус-пилюля

`rounded-full px-2.5 py-1 text-[11px] font-bold`, семантический фон.
Никогда не обводка без фона — только заливка `-soft` + `-ink` текст.

### Бейдж-иконка ч/с, предупреждения

Иконка (lucide) + текст: `inline-flex items-center gap-1 rounded-full
bg-{tone}-soft px-2 py-0.5 text-[11px] font-bold text-{tone}-ink`.

### Список (строка списка клиентов / аренд)

- Слева тонкая полоска-акцент (`before:absolute before:left-0 before:w-[3px]`)
  — красная для ЧС / оранжевая для просрочки / прозрачная для обычных
- Активная строка: `bg-blue-600 text-white shadow-card-lg` (фирменный синий
  фон, белый текст, полупрозрачные аватары `bg-white/20`)
- Фейд-маска по краям прокрутки (утилита `.scroll-fade-y`) — эффект iPhone-picker
- Тонкий кастомный скроллбар (утилита `.scrollbar-thin`)

### Модалки

- Бэкдроп: `fixed inset-0 bg-ink/55 backdrop-blur-sm`
- Контейнер: `rounded-2xl bg-surface shadow-card-lg max-w-[520–780px]`
- Анимации: `animate-backdrop-in` / `animate-modal-in` при открытии,
  `animate-backdrop-out` / `animate-modal-out` при закрытии (160–220мс)
- Esc и клик снаружи закрывают
- Header: `bg-surface-soft`, footer — тоже, для визуального разделения

### Tooltip / toast «Скопировано»

Отдельная обёртка для центрирования + внутренний span с
`animate-toast-in` (только opacity + translateY, без scale — чтобы не
конфликтовать с `translate-x-1/2`).

## 8. Анимации

Keyframes в `tailwind.config.ts`:

| Имя | Длительность | Когда |
|---|---|---|
| `backdrop-in` / `backdrop-out` | 180/160мс | фон модалок |
| `modal-in` / `modal-out` | 220/160мс | сами модалки |
| `toast-in` | 180мс | всплывающие тосты |
| `wave-hand` | 2.5с ∞ | приветствие на дашборде |
| `fly-away` | 500мс | завершение задачи |

Правило: никакой пляски без причины. Анимация должна сигнализировать о
действии (открытие, закрытие, сохранение) — не украшать интерфейс.

## 9. Иконки

- Источник: **lucide-react** (`lucide-react`)
- Размеры: 11–12 (inline в тексте), 13–14 (в кнопках), 16–18 (в title/chip),
  22+ (empty-state)
- Цвет иконок определяется контекстом: `text-blue-600` для акцентных,
  `text-muted-2` для нейтральных, `text-red-ink` / `text-green-ink` для
  семантических
- Эмодзи **не использовать** в UI (кроме демонстрационных примеров /
  empty-state иллюстраций)

## 10. Формы и поля ввода

```tsx
<input
  className="h-9 w-full rounded-[10px] border border-border bg-surface
             px-3 text-[13px] text-ink outline-none
             placeholder:text-muted-2 focus:border-blue-600"
/>
```

- Высота инпута: **36px (h-9)**
- Рамка: `border-border`, на фокусе → `border-blue-600`
- Ошибка: `border-red focus:border-red` + красный текст `text-red-ink` под полем
- Лейбл: `text-[12px] font-semibold text-ink`, Hint: `text-[11px] text-muted-2`

### Сегментированные кнопки (табы-фильтры)

```tsx
<div className="inline-flex rounded-full bg-surface-soft p-0.5">
  <button className={cn(
    "rounded-full px-3 py-1 text-[12px] font-semibold",
    active ? "bg-white text-ink shadow-card-sm" : "text-muted hover:text-ink"
  )} />
</div>
```

### Загрузка файлов

Компоненты: `DocUpload` (1 файл) и `DocUploadMulti` (много с названиями+
комментариями). Поддерживают drag-drop, превью (img), клик по превью
открывает `FilePreviewModal`.

## 11. Навигация

- **Сайдбар** слева, `68px` свёрнут / `232px` развёрнут при hover. В
  свёрнутом — tooltip при наведении на пункт.
- Неготовые разделы — opacity 50% + бейдж `скоро`, disabled.
- Cross-page переходы — через `navigationStore.navigate({ route, clientId,
  rentalId, from })`. Page consume’ит pending-selection в `useEffect` при
  mount (`consumePending(route)`).
- **Заголовок окна Electron** — кастомный `TitleBar` (высота 36px,
  draggable, свои кнопки min/max/close). Тайтлбар показывается только в
  Electron, в вебе его нет.

## 12. Где живут файлы дизайна

```
apps/web/src/
├── index.css                    # токены, keyframes, утилиты
├── tailwind.config.ts           # маппинг токенов в utility-классы, анимации
├── components/ui/               # shadcn-подобные примитивы (button, card)
├── pages/                       # по экранам: dashboard / clients / rentals
│   └── <page>/<Page>.tsx + <Component>.tsx
└── app/
    ├── Sidebar.tsx              # навигация
    ├── TitleBar.tsx             # Electron-заголовок
    └── App.tsx                  # корневая раскладка
```

Конвенция: **по одному компоненту на файл** в `pages/<тема>/`. Хелперы
(функции форматирования, типы) — внутри того же файла, если используются
только там.

## 13. Проверочный чек-лист при новом экране/компоненте

- [ ] Использовал ли токены вместо hex/rgb
- [ ] Заголовок H1/H2 только одного типа (не крупный+uppercase+bold вместе)
- [ ] Цвета семантические (red для проблем, green для нормы, blue для
      действий), **не декоративные**
- [ ] Одна primary-кнопка слева, остальное — в dropdown
- [ ] Статус пилюлей первым, затем сущность
- [ ] Числа в `tabular-nums`
- [ ] Рамок/теней не больше чем нужно
- [ ] Есть пустое состояние (`Empty`) — скелет/текст/подсказка
- [ ] Есть состояние загрузки (если асинхронное)
- [ ] Клавиатурный фокус виден (focus:border-blue-600)
- [ ] Анимация есть только при открытии/закрытии/сохранении

## 14. Принципы UX, которые мы уже зафиксировали

- Телефон клиента — **самое важное** поле в карточке, крупным шрифтом, в
  начале инфо-строки, кликабельный `tel:`
- Рейтинг клиента — **привязан к его имени**, не отдельно
- Активная строка списка — **залита бренд-синим**, чтобы не теряться
  среди цветных подсветок статусов
- Фейд по краям скролла (iPhone-эффект) при прокрутке длинных списков
- Модалки не имеют прыжков: анимация только opacity/translate-y, без
  scale, если элемент уже центрирован через `translate-x-1/2`
- Скутеры в парке — единый массив (`mockPark`), аренды ссылаются на них
  по имени. Клиенты и аренды связаны через `clientId` → единый источник
  правды
